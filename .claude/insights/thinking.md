# Thinking Architecture

This file tracks HOW we think and decide in this project. Each entry captures not just WHAT was decided but WHY.

## Knowledge Type Key

- **Explicit** — Rule-based, can be documented as SOP or automated
- **Implicit** — Can be explained with effort, involves nuance and decision trees
- **Tacit** — Requires intuition, pattern recognition, years of experience
