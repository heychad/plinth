## Part 1: Supabase Schema

### Design Principles

1. Every table that holds tenant-scoped data has `tenant_id` as a foreign key — RLS enforces isolation at the DB layer.
2. Consultant-scoped tables have `consultant_id`. Consultants see all their tenants; tenants see only themselves.
3. Agent configs are data, never code. The runner reads configs; the config makes each client's agent unique.
4. Integration slots are named references — credentials never live in agent configs.
5. The `agent_runs` table is the source of truth for execution state; the UI polls it.

```sql
  SELECT NULLIF(
```

### JWT Custom Claims Hook

To make RLS policies work, custom claims (`consultant_id`, `tenant_id`, `user_role`) must be injected into the JWT at sign-in. Use Supabase's Auth Hook:

```sql
-- Supabase Dashboard → Authentication → Hooks → Custom Access Token Hook
-- Point to this function:

CREATE OR REPLACE FUNCTION public.custom_access_token_hook(event JSONB)
RETURNS JSONB AS $$
DECLARE
  user_record RECORD;
  claims JSONB;
BEGIN
  -- Look up the user's role and IDs
  SELECT u.role, u.tenant_id, u.consultant_id
  INTO user_record
  FROM public.users u
  WHERE u.auth_user_id = (event->>'user_id')::UUID;

  claims := event->'claims';

  IF user_record IS NOT NULL THEN
    claims := jsonb_set(claims, '{user_role}', to_jsonb(user_record.role));

    IF user_record.consultant_id IS NOT NULL THEN
      claims := jsonb_set(claims, '{consultant_id}', to_jsonb(user_record.consultant_id::TEXT));
    END IF;

    IF user_record.tenant_id IS NOT NULL THEN
      claims := jsonb_set(claims, '{tenant_id}', to_jsonb(user_record.tenant_id::TEXT));
    END IF;
  END IF;

  RETURN jsonb_set(event, '{claims}', claims);
END;
$$ LANGUAGE PLPGSQL STABLE;
```

### Middleware

`middleware.ts` runs on every request. It handles:
1. Auth check — redirect unauthenticated users to `/auth/login`
2. Role-based routing — send consultants to `/(consultant)/`, clients to `/(client)/`
3. Theme injection — reads theme from DB (with caching), sets headers for SSR

```typescript
// middleware.ts
import { createServerClient } from '@supabase/ssr'
import { NextResponse, type NextRequest } from 'next/server'

export async function middleware(request: NextRequest) {
  let response = NextResponse.next({
    request: { headers: request.headers },
  })

  const supabase = createServerClient(
    process.env.NEXT_PUBLIC_SUPABASE_URL!,
    process.env.NEXT_PUBLIC_SUPABASE_ANON_KEY!,
    {
      cookies: {
        getAll: () => request.cookies.getAll(),
        setAll: (cookies) => {
          cookies.forEach(({ name, value, options }) => {
            request.cookies.set(name, value)
            response.cookies.set(name, value, options)
          })
        },
      },
    }
  )

  const { data: { user } } = await supabase.auth.getUser()
  const pathname = request.nextUrl.pathname

  // Public routes — no auth required
  const publicRoutes = ['/auth/login', '/auth/callback', '/auth/signout']
  if (publicRoutes.some((r) => pathname.startsWith(r))) {
    return response
  }

  // Not authenticated → redirect to login
  if (!user) {
    const url = request.nextUrl.clone()
    url.pathname = '/auth/login'
    return NextResponse.redirect(url)
  }

  // Fetch JWT claims (role set by custom_access_token_hook)
  const { data: { session } } = await supabase.auth.getSession()
  const claims = session?.access_token
    ? JSON.parse(atob(session.access_token.split('.')[1]))
    : {}

  const userRole = claims.user_role as string | undefined

  // Role-based routing guards
  if (pathname.startsWith('/dashboard') || pathname.startsWith('/clients') || pathname.startsWith('/agents') || pathname.startsWith('/settings')) {
    if (userRole !== 'consultant' && userRole !== 'platform_admin') {
      const url = request.nextUrl.clone()
      url.pathname = '/app'
      return NextResponse.redirect(url)
    }
  }

  if (pathname.startsWith('/app')) {
    if (userRole === 'consultant') {
      const url = request.nextUrl.clone()
      url.pathname = '/dashboard'
      return NextResponse.redirect(url)
    }
  }

  // Root → redirect to correct home based on role
  if (pathname === '/') {
    const url = request.nextUrl.clone()
    url.pathname = userRole === 'consultant' ? '/dashboard' : '/app'
    return NextResponse.redirect(url)
  }

  return response
}

export const config = {
  matcher: ['/((?!_next/static|_next/image|favicon.ico|.*\\.(?:svg|png|jpg|jpeg|gif|webp)$).*)'],
}
```

### Config Loader Pattern

The config loader merges the template's base config with the tenant's customizations, resolves `{{variable}}` substitutions in the system prompt, and returns a ready-to-execute config.

```typescript
// lib/agents/config-loader.ts
import { createClient } from '@/lib/supabase/admin'
import type { AgentConfig, AgentTemplate } from '@/types/database'

export interface ResolvedAgentConfig {
  agentConfig: AgentConfig
  template: AgentTemplate
  resolvedSystemPrompt: string
  model: string
  integrationSlots: string[]
}

const DEFAULT_MODEL = 'claude-sonnet-4-6'

export async function loadAgentConfig(
  agentConfigId: string,
  tenantId: string
): Promise<ResolvedAgentConfig> {
  const supabase = createClient() // service role — bypasses RLS for server jobs

  // Load the agent config (with template joined)
  const { data: agentConfig, error: configError } = await supabase
    .from('agent_configs')
    .select(`
      *,
      agent_templates (*)
    `)
    .eq('id', agentConfigId)
    .eq('tenant_id', tenantId)
    .single()

  if (configError || !agentConfig) {
    throw new Error(`Agent config not found: ${agentConfigId}`)
  }

  const template = agentConfig.agent_templates as AgentTemplate

  // Merge: template defaults → tenant config → resolved
  const mergedConfig = {
    ...template.default_config,
    ...agentConfig.config,
  }

  // Substitute {{variable}} placeholders in system prompt
  const resolvedSystemPrompt = resolvePromptTemplate(
    template.system_prompt,
    mergedConfig
  )

  const model = agentConfig.model_override ?? DEFAULT_MODEL

  return {
    agentConfig,
    template,
    resolvedSystemPrompt,
    model,
    integrationSlots: template.integration_slots,
  }
}

function resolvePromptTemplate(
  template: string,
  config: Record<string, unknown>
): string {
  return template.replace(/\{\{(\w+)\}\}/g, (match, key) => {
    const value = config[key]
    return value !== undefined ? String(value) : match
  })
}
```

### Credential Resolver Pattern

Credentials are resolved from the DB into Composio entity IDs. The entity ID is what gets passed to Composio at runtime — the actual token never enters the application code.

```typescript
// lib/agents/credential-resolver.ts
import { createClient } from '@/lib/supabase/admin'

export interface ResolvedCredentials {
  [slotName: string]: {
    provider: string
    composioEntityId: string | null  // null means not connected
    status: string
  }
}

export async function resolveCredentials(
  tenantId: string,
  slots: string[]
): Promise<ResolvedCredentials> {
  if (slots.length === 0) return {}

  const supabase = createClient()

  const { data: credentials, error } = await supabase
    .from('credentials')
    .select('slot_name, provider, composio_entity_id, status')
    .eq('tenant_id', tenantId)
    .in('slot_name', slots)

  if (error) throw new Error(`Failed to resolve credentials: ${error.message}`)

  const resolved: ResolvedCredentials = {}

  for (const slot of slots) {
    const cred = credentials?.find((c) => c.slot_name === slot)
    resolved[slot] = {
      provider: cred?.provider ?? 'unknown',
      composioEntityId: cred?.composio_entity_id ?? null,
      status: cred?.status ?? 'pending',
    }
  }

  // Validate all required slots are connected
  const missingSlots = slots.filter(
    (slot) => resolved[slot].composioEntityId === null
  )

  if (missingSlots.length > 0) {
    throw new Error(
      `Missing credentials for slots: ${missingSlots.join(', ')}. ` +
      `Client must connect integrations before running this agent.`
    )
  }

  return resolved
}
```

### Generic Agent Runner (Simple / Single-step)

```typescript
// lib/agents/runner.ts
import Anthropic from '@anthropic-ai/sdk'
import { loadAgentConfig } from './config-loader'
import { resolveCredentials } from './credential-resolver'
import { trackUsage } from './usage-tracker'
import { getComposioTools } from '@/lib/composio/tools'

export interface RunAgentOptions {
  agentConfigId: string
  tenantId: string
  consultantId: string
  runId: string
  input: Record<string, unknown>
  stepSlug?: string             // for pipeline steps that call this directly
}

export interface RunAgentResult {
  output: Record<string, unknown>
  tokensIn: number
  tokensOut: number
  costUsd: number
}

const anthropic = new Anthropic({
  apiKey: process.env.ANTHROPIC_API_KEY!,
})

export async function runAgent(options: RunAgentOptions): Promise<RunAgentResult> {
  const { agentConfigId, tenantId, consultantId, runId, input, stepSlug } = options

  // 1. Load config (template + tenant customizations merged)
  const { resolvedSystemPrompt, model, integrationSlots, template, agentConfig } =
    await loadAgentConfig(agentConfigId, tenantId)

  // 2. Resolve credentials (Composio entity IDs per slot)
  const credentials = await resolveCredentials(tenantId, integrationSlots)

  // 3. Get Composio tools for this agent's integration slots
  const tools = await getComposioTools(integrationSlots, credentials)

  // 4. Build user message from input
  const userMessage = buildUserMessage(input, template.input_schema)

  // 5. Execute via Anthropic messages.create()
  //    Agentic loop: if tool use, execute tool and continue until text response.
  const messages: Anthropic.MessageParam[] = [
    { role: 'user', content: userMessage }
  ]

  let tokensIn = 0
  let tokensOut = 0
  let finalOutput: Record<string, unknown> = {}

  while (true) {
    const response = await anthropic.messages.create({
      model,
      max_tokens: 8096,
      system: resolvedSystemPrompt,
      messages,
      tools: tools.length > 0 ? tools : undefined,
    })

    tokensIn += response.usage.input_tokens
    tokensOut += response.usage.output_tokens

    if (response.stop_reason === 'end_turn') {
      // Extract text content as output
      const textContent = response.content
        .filter((b) => b.type === 'text')
        .map((b) => (b as Anthropic.TextBlock).text)
        .join('\n')

      finalOutput = { text: textContent }
      break
    }

    if (response.stop_reason === 'tool_use') {
      // Execute tool calls via Composio
      const toolResults: Anthropic.MessageParam = {
        role: 'user',
        content: [],
      }

      for (const block of response.content) {
        if (block.type === 'tool_use') {
          const result = await executeComposioTool(
            block.name,
            block.input as Record<string, unknown>,
            credentials
          )
          ;(toolResults.content as Anthropic.ToolResultBlockParam[]).push({
            type: 'tool_result',
            tool_use_id: block.id,
            content: JSON.stringify(result),
          })
        }
      }

      messages.push({ role: 'assistant', content: response.content })
      messages.push(toolResults)
      continue
    }

    // max_tokens or other stop — exit loop
    break
  }

  // 6. Track usage
  const costUsd = calculateCost(model, tokensIn, tokensOut)
  await trackUsage({ tenantId, consultantId, agentConfigId, runId, tokensIn, tokensOut, costUsd })

  return { output: finalOutput, tokensIn, tokensOut, costUsd }
}

function buildUserMessage(
  input: Record<string, unknown>,
  schema: Record<string, { label: string }>
): string {
  const parts: string[] = []
  for (const [key, field] of Object.entries(schema)) {
    if (input[key] !== undefined) {
      parts.push(`${field.label}:\n${input[key]}`)
    }
  }
  return parts.join('\n\n---\n\n')
}

function calculateCost(model: string, tokensIn: number, tokensOut: number): number {
  // claude-sonnet-4-6: $3/M in, $15/M out (approximate)
  const rates: Record<string, { in: number; out: number }> = {
    'claude-sonnet-4-6': { in: 3 / 1_000_000, out: 15 / 1_000_000 },
    'claude-opus-4-6': { in: 15 / 1_000_000, out: 75 / 1_000_000 },
  }
  const rate = rates[model] ?? rates['claude-sonnet-4-6']
  return tokensIn * rate.in + tokensOut * rate.out
}
```

```typescript
import { createClient } from '@/lib/supabase/admin'
import { loadAgentConfig } from '@/lib/agents/config-loader'
import { resolveCredentials } from '@/lib/agents/credential-resolver'
import Anthropic from '@anthropic-ai/sdk'

const anthropic = new Anthropic({ apiKey: process.env.ANTHROPIC_API_KEY! })
const supabase = createClient()

export const contentRepurposingFunction = inngest.createFunction(
  {
    id: 'content-repurposing-machine',
    name: 'Content Repurposing Machine',
    retries: 2,
    // Timeout: 30 minutes max (content generation can take 15-20 min)
    timeouts: { finish: '30m' },
  },
  { event: 'agent/run.requested' },
  async ({ event, step }) => {
    const { runId, agentConfigId, tenantId, consultantId, input } = event.data

    // Only handle this specific agent slug
    if (event.data.agentSlug !== 'content-repurposing-machine') return

    // Mark run as running
    await step.run('mark-run-started', async () => {
      await supabase
        .from('agent_runs')
        .update({ status: 'running', started_at: new Date().toISOString() })
        .eq('id', runId)
    })

    // Load config once — reuse across steps
    const resolvedConfig = await step.run('load-config', async () => {
      return loadAgentConfig(agentConfigId, tenantId)
    })

    const credentials = await step.run('resolve-credentials', async () => {
      return resolveCredentials(tenantId, resolvedConfig.integrationSlots)
    })

    // STEP 1: Gather and validate inputs
    const gatheredInputs = await step.run('gather-inputs', async () => {
      await markStepRunning(runId, tenantId, 'gather-inputs', 'Gathering Inputs', 1)

      const response = await anthropic.messages.create({
        model: resolvedConfig.model,
        max_tokens: 2048,
        system: buildStepPrompt(resolvedConfig, 'gather-inputs'),
        messages: [{ role: 'user', content: formatInput(input) }],
      })

      const result = extractTextContent(response)
      await markStepCompleted(runId, tenantId, 'gather-inputs', result, response.usage)
      return result
    })

    // STEP 2: Generate content (the heavy step)
    const generatedContent = await step.run('generate-content', async () => {
      await markStepRunning(runId, tenantId, 'generate-content', 'Generating Content', 2)

      const response = await anthropic.messages.create({
        model: resolvedConfig.model,
        max_tokens: 16384,
        system: buildStepPrompt(resolvedConfig, 'generate-content'),
        messages: [
          { role: 'user', content: formatInput(input) },
          { role: 'assistant', content: `Inputs validated:\n${gatheredInputs}\n\nNow generating content...` },
        ],
      })

      const result = extractTextContent(response)
      await markStepCompleted(runId, tenantId, 'generate-content', result, response.usage)
      return result
    })

    // STEP 3: Quality review
    const reviewedContent = await step.run('review-content', async () => {
      await markStepRunning(runId, tenantId, 'review-content', 'Quality Review', 3)

      const response = await anthropic.messages.create({
        model: resolvedConfig.model,
        max_tokens: 16384,
        system: buildStepPrompt(resolvedConfig, 'review-content'),
        messages: [
          { role: 'user', content: `Review and finalize:\n${generatedContent}` },
        ],
      })

      const result = extractTextContent(response)
      await markStepCompleted(runId, tenantId, 'review-content', result, response.usage)
      return result
    })

    // STEP 4: Write to Google Drive (via Composio)
    const docUrl = await step.run('write-to-google-drive', async () => {
      await markStepRunning(runId, tenantId, 'write-google-drive', 'Writing to Google Drive', 4)

      const entityId = credentials['google_drive'].composioEntityId!

      // Use Composio to create and populate Google Doc
      const { ComposioToolSet } = await import('composio-core')
      const toolset = new ComposioToolSet({ entityId })

      const docResult = await toolset.executeAction({
        action: 'GOOGLEDOCS_CREATE_DOCUMENT',
        params: {
          title: `Weekly Content Plan — ${new Date().toLocaleDateString('en-GB', { day: '2-digit', month: 'short', year: 'numeric' })}`,
          content: reviewedContent,
        },
      })

      const docUrl = docResult?.result?.documentUrl ?? null
      await markStepCompleted(runId, tenantId, 'write-google-drive', { docUrl }, {})
      return docUrl
    })

    // STEP 5: Slack notification
    await step.run('notify-slack', async () => {
      await markStepRunning(runId, tenantId, 'notify-slack', 'Sending Slack Notification', 5)

      const entityId = credentials['slack'].composioEntityId!
      const slackChannel = (resolvedConfig.agentConfig.config as Record<string, string>)['slack_channel_id']

      const { ComposioToolSet } = await import('composio-core')
      const toolset = new ComposioToolSet({ entityId })

      await toolset.executeAction({

        params: {
          channel: slackChannel,
          text: `Your newsletter has been repurposed!\n\nWeekly Content Plan: ${docUrl}\n\n34 pieces of content ready for review.`,
        },
      })

      await markStepCompleted(runId, tenantId, 'notify-slack', { sent: true }, {})
    })

    // Mark run as completed
    await step.run('mark-run-completed', async () => {
      await supabase
        .from('agent_runs')
        .update({
          status: 'completed',
          output: { doc_url: docUrl, content_pieces: 34 },
          completed_at: new Date().toISOString(),
        })
        .eq('id', runId)
    })

    return { success: true, docUrl }
  }

// ── Helpers ──

function buildStepPrompt(config: Awaited<ReturnType<typeof loadAgentConfig>>, stepSlug: string): string {
  const step = config.template.pipeline_steps?.find((s) => s.slug === stepSlug)
  if (!step) return config.resolvedSystemPrompt
  return config.resolvedSystemPrompt + '\n\n---\n\n' + step.prompt_template
}

function formatInput(input: Record<string, unknown>): string {
  return Object.entries(input)
    .map(([k, v]) => `${k}:\n${v}`)
    .join('\n\n---\n\n')
}

function extractTextContent(response: Anthropic.Message): string {
  return response.content
    .filter((b) => b.type === 'text')
    .map((b) => (b as Anthropic.TextBlock).text)
    .join('\n')
}

async function markStepRunning(
  runId: string, tenantId: string, stepSlug: string, displayName: string, order: number
) {
  const supabase = createClient()
  await supabase.from('agent_run_steps').upsert({
    run_id: runId,
    tenant_id: tenantId,
    step_slug: stepSlug,
    step_display_name: displayName,
    step_order: order,
    status: 'running',
    started_at: new Date().toISOString(),
  }, { onConflict: 'run_id,step_slug' })
}

async function markStepCompleted(
  runId: string, tenantId: string, stepSlug: string,
  output: unknown, usage: { input_tokens?: number; output_tokens?: number }
) {
  const supabase = createClient()
  await supabase
    .from('agent_run_steps')
    .update({
      status: 'completed',
      output,
      prompt_tokens: usage.input_tokens ?? 0,
      completion_tokens: usage.output_tokens ?? 0,
      completed_at: new Date().toISOString(),
    })
    .eq('run_id', runId)
    .eq('step_slug', stepSlug)
}
```

### API Route: Enqueue a Run

```typescript
// app/api/agent/run/route.ts
import { NextRequest, NextResponse } from 'next/server'
import { createClient } from '@/lib/supabase/server'
import { inngest } from '@/inngest/client'

export async function POST(request: NextRequest) {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()

  if (!user) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
  }

  const body = await request.json()
  const { agentConfigId, input } = body as {
    agentConfigId: string
    input: Record<string, unknown>
  }

  if (!agentConfigId || !input) {
    return NextResponse.json({ error: 'Missing agentConfigId or input' }, { status: 400 })
  }

  // Look up agent config + validate tenant ownership
  const { data: agentConfig, error } = await supabase
    .from('agent_configs')
    .select('*, agent_templates(slug), tenants(consultant_id)')
    .eq('id', agentConfigId)
    .single()

  if (error || !agentConfig) {
    return NextResponse.json({ error: 'Agent config not found' }, { status: 404 })
  }

  // Create the run row (queued state)
  const { data: run, error: runError } = await supabase
    .from('agent_runs')
    .insert({
      agent_config_id: agentConfigId,
      tenant_id: agentConfig.tenant_id,
      status: 'queued',
      trigger_type: 'manual',
      triggered_by: user.id,
      input,
    })
    .select()
    .single()

  if (runError || !run) {
    return NextResponse.json({ error: 'Failed to create run' }, { status: 500 })
  }

  // Enqueue Inngest event
  await inngest.send({
    name: 'agent/run.requested',
    data: {
      runId: run.id,
      agentConfigId,
      tenantId: agentConfig.tenant_id,
      consultantId: agentConfig.tenants.consultant_id,
      agentSlug: agentConfig.agent_templates.slug,
      input,
      triggeredBy: user.id,
    },
  })

  return NextResponse.json({ runId: run.id })
}
```

```typescript
// app/api/agent/[runId]/status/route.ts
import { NextRequest, NextResponse } from 'next/server'
import { createClient } from '@/lib/supabase/server'

export async function GET(
  _request: NextRequest,
  { params }: { params: { runId: string } }
) {
  const supabase = await createClient()
  const { data: { user } } = await supabase.auth.getUser()

  if (!user) {
    return NextResponse.json({ error: 'Unauthorized' }, { status: 401 })
  }

  const { data: run } = await supabase
    .from('agent_runs')
    .select(`
      id, status, output, error_message,
      queued_at, started_at, completed_at, duration_ms,
      total_tokens_in, total_tokens_out, total_cost_usd,
      agent_run_steps (
        step_slug, step_display_name, step_order,
        status, started_at, completed_at, output
      )
    `)
    .eq('id', params.runId)
    .single()

  if (!run) {
    return NextResponse.json({ error: 'Run not found' }, { status: 404 })
  }

  return NextResponse.json(run)
}
```

## Rollback Plan

If the Supabase migration causes issues:
```bash
# Revert via Supabase CLI
supabase db reset   # drops and recreates from migrations

# Or manually drop specific tables
DROP TABLE IF EXISTS agent_run_steps CASCADE;
DROP TABLE IF EXISTS agent_runs CASCADE;
# etc. in reverse dependency order
```

If Inngest functions break a run:
- Runs table `status = 'failed'` captures error state
- Failed runs can be retried via Inngest dashboard
- The DB row is the source of truth — Inngest is stateless

## Open Questions

| # | Question | Impact | Decision Needed By |
|---|----------|--------|-------------------|
| 1 | Does Maeve want a separate auth account for her "client view" or should one login let her toggle between consultant/client views? | Auth model + middleware routing complexity | Before Step 4 |
| 2 | Should theme colors affect the consultant dashboard too, or only the client portal? | ThemeProvider scope | Before Step 7 |
| 3 | Composio starter plan is $49/month for 5,000 actions. At 3 agents × 3 tenants × weekly runs = ~36 actions/month currently. When do we upgrade? | Budget planning | Phase 2 concern |
| 4 | For the demo, should seeded run history show real token counts or just placeholder numbers? | Seed script values | Before Step 5 |
| 5 | Is the Supabase hook approach for JWT claims the right pattern, or should we use a separate user metadata lookup on each request? | Performance + RLS design | Before Step 6 |
| 6 | Should Maeve's BOTH roles (consultant + her own client) be handled by one user record with a toggle in UI, or two separate user records? | Data model + UX | Before Step 5 |

**Recommendation on Q6:** One auth user (Maeve's email), one `users` row with `role = 'consultant'`. The consultant dashboard has a "View as [tenant name]" switcher that passes a `?tenantId=` param and renders the client portal layout for that tenant. This is simpler than two separate user records and matches how Maeve would actually use it (she IS the consultant, occasionally previewing what her clients see).
